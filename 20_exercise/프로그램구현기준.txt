1. 파일 관리
   - html, css, javascript 를 각각의 파일로 분리해서 작성할 것.

2. 구조성
   - 비즈니스 로직, 예외 처리, UI 처리는 각각 분리하여 구현할 것.
     그렇다고, javascript 파일을 모듈로 관리하는 것은 아님.
     함수 단위로 각각 분리하여 구현하면 됨.

3. 재사용성
   - 기능의 경우 하나의 함수로 통일하여 구현해서 재사용성을 높일 것.
   - 데이터의 경우 배열 등으로 구현하여 재사용성을 높일 것.

4. 확장성
   - 현금 버튼, 음료수 버튼이 새롭게 추가가가 되더라도 
     큰 수정없이 프로그램에 반영이 되도록 데이터로 관리할 것.
   - 윈도우가 로딩시 새롭게 추가가 된 부분이 UI에 반영이
     되도록 할 것.
